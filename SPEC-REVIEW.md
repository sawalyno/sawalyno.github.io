## 総評

コンセプト（ダーク × Windows XP/7 × ターミナル）が明確で、ジェネリックな AI 風サイトに陥らない強い視点を持っている。フォント選択（JetBrains Mono / Hanken Grotesk / Noto Sans JP）、ドミナント＋アクセント方針のカラーパレット、控えめで意図的なアニメーションは良好。Play Console 審査を主目的とする最小構成としても十分機能する。

一方で、(a) ターミナル内に日本語混在文を置く実装上の落とし穴、(b) JSON-LD Organization など法人実在性をさらに補強する SEO メタの欠落、(c) アプリカードのストアバッジ運用未定義、(d) フォーカスリングや `_ □ ×` ボタンの ARIA 扱いなど A11y の細部、(e) ブレークポイント単一による中間幅の間延び、あたりに詰めの余地がある。下記レビュー表のうち「重要度：高」項目は公開前に解消を推奨する。

## 観点別サマリ

### 法務・運営者証明
- **R-04** が最重要。サイト本体には法人併記のみ、実値はストア提出フォームでのみ扱う運用に絞ると、後の運用負荷とリスクが下がる。
- **R-22** はストア審査の最初のレビュアー印象に直結する。Developer Name とサイト主体名の対応表は早めに固める。

### デザイン（frontend-design 観点）
- 強み：明確な POV、ドミナント＋鋭いアクセント、控えめなモーション、CJK を含む 3 フォント編成。
- 弱み：上下双子のクロム要素（**R-06**）、薄すぎるテクスチャ（**R-07**）、青の効かせどころが不足（**R-19**）。
- **コンセプトの「決め所」は1〜2箇所に集中**させると XP/7 の懐かしさが立ち上がる。今の仕様は均等に薄く広げているため、印象が中庸に寄りやすい。

### 実装上の落とし穴
- **R-01**（CJK 等幅問題）は実装してから気付くと手戻りが大きい。先に「ターミナル内は英語のみ」と仕様で確定させるのが楽。

### A11y / SEO
- **R-02**（JSON-LD Organization）は法人実在性証明という主目的に直結するため最優先で追加。
- **R-09**（`_ □ ×` の `aria-hidden`）と **R-10**（focus-visible の色）は実装段階で抜けやすいので仕様に書いておく。

### 公開前 必須対応（重要度：高 のみ抜粋）
- R-01：ターミナル内 CJK の扱いを仕様で確定
- R-02：JSON-LD Organization を §9 に追加
- R-04：所在地非掲載に関する文言をサイト HTML から外す方針を確定
- R-09：`_ □ ×` の装飾扱い ARIA 仕様
